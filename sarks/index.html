Cloud-Native CI/CD

Red Hat OpenShift Pipelines is based on Kubernetes resources and uses Tekton building blocks to automate deployments across multiple platforms. It abstracts away underlying implementation details, allowing for portable pipeline definitions.

Core Concepts

Understanding the building blocks of OpenShift Pipelines.

Tasks

Tasks are the fundamental building blocks of a Pipeline. They consist of a sequence of steps that are executed in order. Each step runs in its own container and achieves a specific goal, such as compiling code, running tests, or building an image.

Pipelines

A Pipeline is a collection of Task resources arranged in a specific order of execution. You can define dependencies between tasks, pass parameters, and share workspaces to create complex workflows that automate your entire software delivery process.

Triggers

Triggers enable your pipelines to respond to external events. They capture events like Git push or pull requests, extract relevant information (like commit IDs), and automatically instantiate a new PipelineRun to process the changes.

Pipeline Elements

Behind the scenes of execution.

TaskRun

A TaskRun represents a single execution of a Task. It binds the Task to specific inputs, outputs, and execution parameters. You can create a TaskRun manually or it can be created automatically by a PipelineRun.

PipelineRun

A PipelineRun is the running instance of a Pipeline. It initiates the execution of all Tasks defined in the Pipeline, managing the order and passing data between them. It provides a record of the execution history.

Trigger Resources

The Trigger system consists of TriggerTemplates (blueprints for resources), TriggerBindings (extracting fields from events), and EventListeners (listening for events) to create a robust CI/CD automation.
